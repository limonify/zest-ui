`Radio` is a single choice within a `RadioGroup`. A standalone radio has no source of truth, so `Radio.Root` requires a group and throws without one.

## Usage

```tsx
import { Text } from 'react-native';
import { RadioGroup, Radio } from '@limonify/zest';

<RadioGroup value={value} onValueChange={setValue}>
  {options.map((o) => (
    <Radio.Root key={o} value={o} style={styles.radio}>
      <Radio.Indicator style={styles.dot} />
      <Text>{o}</Text>
    </Radio.Root>
  ))}
</RadioGroup>
```

## API

`Radio` — the props below sit on the root. Every part also accepts the [common props](/docs/philosophy) (`style`/`className` as a value or function of state, `render`, `ref`).

## Accessibility

`Radio.Root` is `accessibilityRole="radio"` with `accessibilityState.checked`. `Radio.Indicator` supports `keepMounted` + `transitionStatus` for exit animations.
