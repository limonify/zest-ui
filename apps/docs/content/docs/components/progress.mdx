---
title: Progress
description: A bar that tracks a task towards completion.
---

`Progress` shows determinate or indeterminate progress. It publishes `status` (`indeterminate | progressing | complete`) and the fill percentage on state; `value={null}` is indeterminate.

## Usage

The behaviour is headless — the **Code** tab is the component; the **StyleSheet** and **Uniwind** tabs are two ways to design it.

<Tabs items={['Code', 'StyleSheet', 'Uniwind']}>

<Tab value="Code">

```tsx
import { Progress } from '@limonify/zest-ui';

<Progress.Root value={value}>
  <Progress.Label style={styles.label}>Uploading</Progress.Label>
  <Progress.Value style={styles.label} />
  <Progress.Track style={styles.track}>
    <Progress.Indicator style={(s) => [styles.fill, s.status === 'complete' && styles.done]} />
  </Progress.Track>
</Progress.Root>
```

</Tab>

## Animation

`Progress.Indicator` sizes itself to the current value. To ease the fill smoothly between values, drive a shared value from your own `value` and render the width yourself (percent = `value / max`):

```tsx
import Animated, { useAnimatedStyle } from 'react-native-reanimated';
import { useAnimatedValue } from './useAnimatedValue'; // small helper — see the guide
import { Progress } from '@limonify/zest-ui';

function Fill({ value }: { value: number }) {
  const w = useAnimatedValue(value, { duration: 300 });
  const style = useAnimatedStyle(() => ({ width: `${w.value}%` }));
  return <Progress.Indicator render={(props) => <Animated.View {...props} style={[styles.fill, style]} />} />;
}

<Progress.Root value={value}>
  <Progress.Track style={styles.track}>
    <Fill value={value} />
  </Progress.Track>
</Progress.Root>
```

Skip the animation while indeterminate (`value == null`) — there is no width to ease toward. See the [Reanimated guide](/docs/guides/animation-reanimated) for the full contract.

## Accessibility

`accessibilityRole="progressbar"` with `accessibilityValue` (min/max/now/text). While indeterminate no width is applied — the appearance is yours (usually an animation).
