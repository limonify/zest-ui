`Menu` is an anchored list of actions. It supports submenus (each a nested `Modal`), checkbox and radio items, groups and separators.

<Demo>Opening a menu with a submenu and a radio group.</Demo>

## Usage

The behaviour is headless — the **Code** tab is the component; the **StyleSheet** and **Uniwind** tabs are two ways to design it.

<Tabs items={['Code', 'StyleSheet', 'Uniwind']}>

<Tab value="Code">

```tsx
import { Text } from 'react-native';
import { Menu } from '@limonify/zest';

<Menu.Root>
  <Menu.Trigger style={styles.button}><Text style={styles.buttonText}>Open menu</Text></Menu.Trigger>
  <Menu.Portal>
    <Menu.Backdrop style={styles.scrim} />
    <Menu.Positioner side="bottom" align="start" sideOffset={4}>
      <Menu.Popup style={styles.popup}>
        <Menu.Item onPress={rename} style={styles.item}><Text>Rename</Text></Menu.Item>
        <Menu.Item onPress={remove} style={styles.item}><Text style={styles.danger}>Delete</Text></Menu.Item>
      </Menu.Popup>
    </Menu.Positioner>
  </Menu.Portal>
</Menu.Root>
```

</Tab>

## Accessibility

Each menu level is its own `Modal`, so **every level that should dismiss on an outside tap needs its own `Menu.Backdrop`**. Items close the menu on press by default (`closeOnPress`).
