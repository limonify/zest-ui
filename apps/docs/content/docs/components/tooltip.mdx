title: Tooltip

`Tooltip` is an anchored hint. A touch screen has no hover, so it opens on press rather than hover — there is no hover-delay provider.

## Usage

The behaviour is headless — the **Code** tab is the component; the **StyleSheet** and **Uniwind** tabs are two ways to design it.

<Tabs items={['Code', 'StyleSheet', 'Uniwind']}>

<Tab value="Code">

```tsx
import { Text } from 'react-native';
import { Tooltip } from '@limonify/zest-ui';

<Tooltip.Root>
  <Tooltip.Trigger style={styles.iconButton}><Text>?</Text></Tooltip.Trigger>
  <Tooltip.Portal>
    <Tooltip.Positioner side="top" sideOffset={6}>
      <Tooltip.Popup style={styles.tip}>
        <Tooltip.Arrow style={styles.arrow} />
        <Text style={styles.tipText}>Copy link</Text>
      </Tooltip.Popup>
    </Tooltip.Positioner>
  </Tooltip.Portal>
</Tooltip.Root>
```

</Tab>

## Animation

`Tooltip.Popup` publishes `state.transitionStatus`. Keep it mounted with `keepMounted` and fade it in and out:

```tsx
import Animated, { FadeIn, FadeOut } from 'react-native-reanimated';
import { Tooltip } from '@limonify/zest-ui';

<Tooltip.Popup keepMounted render={(props) => (
  <Animated.View {...props} entering={FadeIn.duration(100)} exiting={FadeOut.duration(100)} />
)}>
  {/* ... */}
</Tooltip.Popup>
```

See the [Reanimated guide](/docs/guides/animation-reanimated) for the full contract.

## Accessibility

The popup is `accessibilityRole="tooltip"`. Upstream's hover-delay `Tooltip.Provider` is not ported — there is no hover on touch.
