`Toast` is the one popup that is **not** a Modal — the app underneath must stay usable. Place `Toast.Provider` at the app root with a `Toast.Viewport`, and raise toasts imperatively with `useToastManager()`.

## Usage

The behaviour is headless — the **Code** tab is the component; the **StyleSheet** and **Uniwind** tabs are two ways to design it.

<Tabs items={['Code', 'StyleSheet', 'Uniwind']}>

<Tab value="Code">

```tsx
import { Toast } from '@limonify/zest-ui';

// 1. At the app root:
<Toast.Provider timeout={4000} limit={3}>
  <App />
  <Toast.Viewport style={styles.viewport}>
    {toasts.map((toast) => (
      <Toast.Root key={toast.id} toast={toast} style={styles.toast}>
        <Toast.Title style={styles.title} />
        <Toast.Description style={styles.desc} />
      </Toast.Root>
    ))}
  </Toast.Viewport>
</Toast.Provider>

// 2. Anywhere inside:
const { add } = Toast.useToastManager();
add({ title: 'Saved', description: 'Your changes are safe.' });
```

</Tab>

## Animation

`Toast.Root` publishes `state.transitionStatus`, its stacking `offsetY`/`index`, and `swipeMovement`. Set `removeOnClose={false}` so the row survives its exit, remove it from the queue when the exit finishes, and use `LinearTransition` so the stack reflows smoothly:

```tsx
import Animated, { SlideInDown, SlideOutDown, LinearTransition, runOnJS } from 'react-native-reanimated';
import { Toast } from '@limonify/zest-ui';

function ToastItem({ toast }) {
  const { remove } = Toast.useToastManager();
  return (
    <Toast.Root
      toast={toast}
      removeOnClose={false}
      render={(props) => (
        <Animated.View
          {...props}
          entering={SlideInDown}
          exiting={SlideOutDown.withCallback(() => runOnJS(remove)(toast.id))}
          layout={LinearTransition}
        />
      )}
    >
      {/* Title / Description */}
    </Toast.Root>
  );
}
```

See the [Reanimated guide](/docs/guides/animation-reanimated) for the full contract.

## Accessibility

The viewport uses `pointerEvents="box-none"` so touches pass through to the app; only the toasts are interactive. `accessibilityLiveRegion="polite"` announces new toasts.
