---
title: Alert Dialog
description: A modal dialog that demands a deliberate choice and ignores outside dismissal.
---

`AlertDialog` is a `Dialog` variant for destructive or blocking confirmations: it does **not** close on an outside press or the back button — the user must pick an action.

## Usage

The behaviour is headless — the **Code** tab is the component; the **StyleSheet** and **Uniwind** tabs are two ways to design it.

<Tabs items={['Code', 'StyleSheet', 'Uniwind']}>

<Tab value="Code">

```tsx
import { Text } from 'react-native';
import { AlertDialog } from '@limonify/zest';

<AlertDialog.Root>
  <AlertDialog.Trigger style={styles.button}><Text style={styles.buttonText}>Delete</Text></AlertDialog.Trigger>
  <AlertDialog.Portal>
    <AlertDialog.Backdrop style={styles.backdrop} />
    <AlertDialog.Viewport style={styles.viewport}>
      <AlertDialog.Popup style={styles.card}>
        <AlertDialog.Title style={styles.title}>Delete file?</AlertDialog.Title>
        <AlertDialog.Description style={styles.body}>This can't be undone.</AlertDialog.Description>
        <AlertDialog.Close style={styles.button}><Text style={styles.buttonText}>Cancel</Text></AlertDialog.Close>
      </AlertDialog.Popup>
    </AlertDialog.Viewport>
  </AlertDialog.Portal>
</AlertDialog.Root>
```

</Tab>

## Accessibility

`accessibilityRole="alertdialog"`. Outside-press and back-button dismissal are disabled — provide an explicit `AlertDialog.Close`.
