`Autocomplete` is a `Combobox` in free-text mode: the typed string **is** the value, and choosing a suggestion fills the input rather than selecting a bound value. It reuses every combobox part.

## Usage

The behaviour is headless — the **Code** tab is the component; the **StyleSheet** and **Uniwind** tabs are two ways to design it.

<Tabs items={['Code', 'StyleSheet', 'Uniwind']}>

<Tab value="Code">

```tsx
import { Text } from 'react-native';
import { Autocomplete } from '@limonify/zest';

<Autocomplete.Root items={TAGS} inputValue={tag} onInputValueChange={setTag}>
  <Autocomplete.Input placeholder="Add a tag" style={styles.input} />
  <Autocomplete.Portal>
    <Autocomplete.Positioner>
      <Autocomplete.Popup style={styles.popup}>
        <Autocomplete.List>
          {(item) => (
            <Autocomplete.Item key={String(item.value)} item={item} style={styles.item}>
              <Text>{item.label}</Text>
            </Autocomplete.Item>
          )}
        </Autocomplete.List>
      </Autocomplete.Popup>
    </Autocomplete.Positioner>
  </Autocomplete.Portal>
</Autocomplete.Root>
```

</Tab>

## Animation

`Autocomplete.Popup` publishes `state.transitionStatus`. Keep it mounted with `keepMounted` and fade the list in and out:

```tsx
import Animated, { FadeIn, FadeOut } from 'react-native-reanimated';
import { Autocomplete } from '@limonify/zest';

<Autocomplete.Popup keepMounted render={(props) => (
  <Animated.View {...props} entering={FadeIn.duration(120)} exiting={FadeOut.duration(120)} />
)}>
  {/* ... */}
</Autocomplete.Popup>
```

See the [Reanimated guide](/docs/guides/animation-reanimated) for the full contract.
