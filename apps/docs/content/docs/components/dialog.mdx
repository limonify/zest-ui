`Dialog` is a modal overlay. Its `Portal` is a React Native `Modal`, so the popup renders above the
app with focus containment and Android back-button dismissal handled for you.

<Demo>Opening and dismissing a dialog.</Demo>

<Callout type="info">
  **Dialog vs AlertDialog vs Drawer.** Use **Dialog** for general modal content;
  **[AlertDialog](/docs/components/alert-dialog)** for a blocking confirm that must not be dismissed
  by an outside tap; **[Drawer](/docs/components/drawer)** for a swipeable sheet anchored to a screen
  edge.
</Callout>

## Usage

The behaviour is headless — the **Code** tab is the component; the **StyleSheet** and **Uniwind** tabs are two ways to design it.

## Dismissal

The `Backdrop` (or a press outside the `Popup` in the `Viewport`) closes the dialog with the
`outside-press` reason; the Android back button and the OS close it with `escape-key`. Every close
reports through `onOpenChange(open, eventDetails)` so you can veto it with `eventDetails.cancel()`.

## Animation

`Dialog.Popup` and `Dialog.Backdrop` publish `state.transitionStatus`. Keep them mounted with `keepMounted` so Reanimated's `exiting` preset has time to play the close:

```tsx
import Animated, { FadeIn, FadeOut } from 'react-native-reanimated';
import { Dialog } from '@limonify/zest-ui';

<Dialog.Backdrop keepMounted render={(props) => (
  <Animated.View {...props} entering={FadeIn.duration(150)} exiting={FadeOut.duration(150)} />
)} />
<Dialog.Popup keepMounted render={(props) => (
  <Animated.View {...props} entering={FadeIn.duration(150)} exiting={FadeOut.duration(150)} />
)}>
  {/* ... */}
</Dialog.Popup>
```

See the [Reanimated guide](/docs/guides/animation-reanimated) for the full contract.

## Accessibility

The popup is `accessibilityViewIsModal` with `accessibilityLabelledBy` pointing at `Dialog.Title`
and `accessibilityDescribedBy` at `Dialog.Description`. Unlike the web version there is no
`aria-modal`-only fallback — the RN `Modal` contains focus natively.
