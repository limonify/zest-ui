`ContextMenu` is a `Menu` opened by a **long press** instead of a tapped trigger, anchored to where your finger landed. It reuses every menu part.

<Demo>Long-pressing a card to open a menu at the touch point.</Demo>

## Usage

The behaviour is headless — the **Code** tab is the component; the **StyleSheet** and **Uniwind** tabs are two ways to design it.

<Tabs items={['Code', 'StyleSheet', 'Uniwind']}>

<Tab value="Code">

```tsx
import { Text } from 'react-native';
import { ContextMenu } from '@limonify/zest-ui';

<ContextMenu.Root>
  <ContextMenu.Trigger style={styles.card}><Text>Long-press me</Text></ContextMenu.Trigger>
  <ContextMenu.Portal>
    <ContextMenu.Backdrop style={styles.transparent} />
    <ContextMenu.Positioner>
      <ContextMenu.Popup style={styles.popup}>
        <ContextMenu.Item onPress={copy} style={styles.item}><Text>Copy</Text></ContextMenu.Item>
        <ContextMenu.Item onPress={remove} style={styles.item}><Text style={styles.danger}>Delete</Text></ContextMenu.Item>
      </ContextMenu.Popup>
    </ContextMenu.Positioner>
  </ContextMenu.Portal>
</ContextMenu.Root>
```

</Tab>

<Tab value="StyleSheet">

## Animation

`ContextMenu.Popup` publishes `state.transitionStatus`. Keep it mounted with `keepMounted` and fade it in and out:

```tsx
import Animated, { FadeIn, FadeOut } from 'react-native-reanimated';
import { ContextMenu } from '@limonify/zest-ui';

<ContextMenu.Popup keepMounted render={(props) => (
  <Animated.View {...props} entering={FadeIn.duration(120)} exiting={FadeOut.duration(120)} />
)}>
  {/* ... */}
</ContextMenu.Popup>
```

See the [Reanimated guide](/docs/guides/animation-reanimated) for the full contract.

## Accessibility

Upstream opens on right-click or long press; a touch screen only has the long press. Everything else matches `Menu`.
