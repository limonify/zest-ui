**zest** (`@limonify/zest-ui`) is a headless component library for React Native: behaviour,
state and accessibility with **zero styling**. Its API is referenced from
[MUI Base UI](https://base-ui.com) — descriptive, composable part names like `Dialog.Popup` and
`Dialog.Backdrop`, so a `Root` and its parts read the same on every component.

## Why headless

Every part renders a plain React Native primitive (`View`, `Pressable`, `Text`, `TextInput`) and
takes your `style`. State reaches your styles through a function:

```tsx
<Checkbox.Root
  style={(state) => [styles.box, state.checked && styles.boxChecked]}
>
  <Checkbox.Indicator>
    <Text>✓</Text>
  </Checkbox.Indicator>
</Checkbox.Root>
```

No theme, no CSS-in-JS, no animation library. You own the look; zest owns the behaviour.

## What's inside

32 components across forms, overlays, feedback and disclosure — everything that has meaning on a
touch screen. Start with [Installation](/docs/installation), skim the shared
